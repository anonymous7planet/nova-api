# Matching Survey (맞선 모드 설문 프로세스 규격)

## 목적
- 맞선 회원의 심리·선호 데이터를 수집하여 매칭 정밀도를 높인다.

## 대상 / 주기
- 대상: **맞선 모드에 가입된 회원**만 참여 가능
- 주기: 매일 **13:00(오후1시)** 기준으로 설문 시작, 24시간 동안 유효
- 참여 제한: **해당 24시간 중 해당 회원의 메인 최초 접속 시 1회만 진행** (완료 후 수정 불가)

## 문항 유형
- 객관식 4지선다 (주요), 5점 척도(선택 가능), 주관식(운영자 검토용)
- 문항 수: 10문항(기본) — 관리자에서 조정 가능

## UX 플로우
1. 맞선 회원이 13:00 이후 메인 페이지 최초 진입 → 팝업으로 설문 알림
2. 사용자가 [설문 시작] 버튼 클릭 → 설문 진행 화면(페이지네이션)
3. 설문 완료 → 감사 화면 및 "오늘 매칭 반영됨" 토스트
4. 미응답 상태: 팝업 닫기 가능 (비참여로 기록, 다음날 재시도)

## 서버/DB 동작
- 설문 응답은 `survey_response` 테이블에 저장 (user_id, question_id, answer_id, created_at)
- 설문은 매일 새로운 `survey_version`으로 관리 (운영자는 새 버전 publish 가능)
- 응답 후 즉시 매칭 스코어 재계산 트리거(비동기 작업 큐로 처리)

## 에지 케이스
- 오프라인 상태에서 작성 → 로컬 캐싱 후 복구 시 전송 (만약 최초 접속 규칙을 우회하는지 검증 필요)
- 설문 항목 삭제/수정 시, 기존 응답과의 호환성 버전 관리 필요

## 개인정보 / 보안
- 설문 답변은 민감할 수 있음 → 익명화/암호화 저장 옵션 고려
- 관리자만 집계 결과 조회 가능 (개인 식별 정보 노출 불가)

## 관리자 기능
- 문항 추가/삭제/순서 변경
- 설문 버전 롤아웃(특정 기간에만 사용)
- 문항별 통계(응답율, 상관관계)
