# NOVA 프로젝트 기획 요약

## 1. 프로젝트 개요
- **프로젝트명:** NOVA  
- **목표:** 온라인 기반의 친구 찾기 + 맞선 시스템을 통한 신뢰도 높은 만남 서비스 구축  
- **서비스 형태:**  
  - 사용자: 모바일 앱 중심  
  - 관리자: 웹 관리 시스템  
  - 일부 관리자 메뉴는 반응형(P+M) 지원

---

## 2. 회원 구조

### 사용자 구분
| 구분 | 설명 |
|------|------|
| 친구모드 | 일반 친구 찾기용 |
| 맞선모드 | 신뢰 기반 맞선용, 관리자 검수 필요 |

---

### 회원 상태 Enum 예시
```java
public enum UserStatus {
    ACT("ACT", "활성 회원"),
    INA("INA", "비활성 회원"),
    SUS("SUS", "정지 회원"),
    WTH("WTH", "탈퇴 회원");
}
```

---

### 회원 등급 Enum 예시
```java
public enum UserLevel {
    BAS("BAS", "일반회원"),
    PRE("PRE", "프리미엄회원"),
    VIP("VIP", "VIP회원");
```

## 3. 관리자 구조

### 관리자 권한 Enum 예시
```java
public enum AdminRole {
    SA("SA", "최고 관리자"),
    MM("MM", "매칭 관리자"),
    CM("CM", "콘텐츠 관리자"),
    SM("SM", "고객지원 관리자");
}
```

---

### 관리자 계정 정책
- 관리자와 일반 사용자 DB 테이블 분리 (`admin`, `user`)
- 관리자는 사용자 앱에서 로그인 가능  
- 로그인 시 인가 정책:  
  - 일반 사용자 → `ROLE_USER`  
  - 관리자 → `ROLE_ADMIN` (세분화 가능)  
- `auth_role` 컬럼 기준으로 접근 제어 수행

## 4. 주요 기능 요약

### 사용자 주요 메뉴 (P+M)
- 회원가입 / 로그인  
- 친구모드 / 맞선모드 전환  
- 일일 선호도 설문 (오후 1시~24시간)  
- 매칭 리스트 및 매칭 확률 표시  
- 채팅 / 알림 / 마이페이지  
- 이상형 등록 및 수정  
- 질문 항목 등록 (상대에게 질문하고 싶은 항목 추가)

---

### 관리자 주요 메뉴 (P)
- 로그인 / 로그 관리  
- 대시보드 (통계 및 주요 지표)  
- 회원 관리 (활동내역, 제재, 상세정보)  
- 매칭 관리 (자동 / 수동 매칭)  
- 매칭 문항 관리  
- 설문 관리  
- 통계 관리 (가입자 수, 성비, MBTI, 지역별 등)  
- 운영 로그 관리  
- 시스템 설정 (언어, 국가, 공지, 약관 등)

---

## 5. 매칭 알고리즘 개요
- **기준 요소:** 나이, 지역, MBTI, 취미, 가치관, 생활습관 등  
- **매칭 로직:**  
  1. 각 항목마다 가중치 부여  
  2. 항목별 유사도 계산 (Cosine Similarity 등)  
  3. 전체 합산하여 1~100% 매칭 점수 산출  
- **이상형 조건 반영:**  
  - 사용자의 실제 데이터 + 선호 이상형 데이터 모두 반영  
  - 예: “나는 흡연하지만 상대는 비흡연을 원함” 같은 조합 처리 가능

## 6. 정책 요약
- **매칭 시간:** 하루 1회 자동 갱신  
- **매칭 제한:** 하루 최대 3명  
- **설문:** 하루 10문항, 최초 접속 시 1회만 응답 가능  
- **매칭 실패 시:** 자동 재추천 (유사도 80% 이상 우선)  
- **데이터 보안:**  
  - 비식별화 데이터 활용  
  - 개인정보 3개월 비활성 후 삭제
